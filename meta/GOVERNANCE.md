# Words of Istanbul: High-Density Governance

This document defines the technical and philosophical standards for the Words of Istanbul repository. All contributions must adhere to these "Meta-Engineering" principles to maintain the project's premium status.

## 1. Quality Standards (Shelby-Standard)

- **High-Density Only:** Do not add filler content. Every entry must provide a significant psychological, historical, or strategic insight.
- **Structural Integrity:** Follow the existing directory structure. Core analytical pieces go to `core/`, while raw data goes to `archive/`.
- **Linguistic Precision:** Turkish content must use sophisticated, high-level vocabulary. Technical terms (Psychogeography, Dérive, etc.) should be used correctly within their theoretical contexts.

## 2. Technical Standards (Intelligence Engine)

- **Structured Metadata:** Any new quote added to the archive must be included in `archive/quotes.json` with the following mandatory fields:
    - `id`: Unique identifier (e.g., q9, q10).
    - `content`: The quote text.
    - `author`: The source.
    - `category`: The thematic area.
    - `mood`: Emotional frequency.
    - `strategic_value`: Technical impact level (1-10).
    - `psychological_weight`: Emotional impact level (1-10).
- **CLI Compatibility:** Ensure that any changes to the JSON structure do not break the `istanbul.py` engine.

## 3. Aesthetic Sovereignty

- **Formatting:** Use consistent markdown formatting. Headers (H1, H2, H3), blockquotes for emphasis, and tables for comparative data.
- **Visuals:** All visual assets must be stored in `assets/` and follow a high-fidelity, cinematic style.

## 4. Philosophical Alignment

This repository is not just a library; it is a tool for understanding **Urban Chaos** and **The Silent Hell**. Contributions should reflect a deep understanding of Istanbul as a complex, living organism.
